# iTools 项目规范（开发前必读）

本项目为 iTools（Tauri 2 + Rust + 原生 TS）。参与本项目开发**必须**遵守 [`doc/开发准则.md`](doc/开发准则.md)。

## 最高红线：诚信实现（杜绝夸大 / 虚假实现）

- **真实实现优先**：不得用空桩、假返回、硬编码假数据冒充已实现的功能。
- **UI 与实现一致**：界面控件的行为必须与后端真实能力一致，不得有「看着能用、点了不生效」的控件。
- **未完成必须诚实标注**：占位 / 桩 / 仅本地 / 未接后端，必须在 UI 明确告知用户，或禁用 / 隐藏；不得让用户误以为可用。
- **如实汇报**：交付时如实区分「真实现 / 桩 / 未验证」，不得把「写了代码」说成「功能可用」。

## 强制审查机制

- **每次功能实现完成后、每次发布前**，必须按 `doc/开发准则.md` 的「发布前审查清单」逐项核对。
- 审查须**追全链路**（UI → invoke → 后端实现体）并采取**对抗性视角**，主动查找空桩 / 假返回 / 不生效控件 / 接口不一致；判为「欺骗性桩」或「虚假」的必须修复后才能发布。

## 安全红线

- 凭据（token / 密钥 / 密码）只从**环境变量**读取，源码与二进制零明文。
- 不硬编码个人 / 敏感数据；账号类默认值为空；服务端地址可配置、不写死；Release 版不产调试日志。

> 详细条款、审查清单与判定标准见 `doc/开发准则.md`。
