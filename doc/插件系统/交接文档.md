# iTools 插件系统（首期）— 交接文档

> 状态：**已实现 + 两轮多维对抗审查 + 全部修复**。cargo check 0 警告 / cargo test 13 通过 / tsc 0 错误 / debug exe 打包成功。
> 定位：**页面插件（给人用）：关键词触发 → 打开 HTML 面板，由 AI 外部生成**放进项目 `plugins/` 目录自动加载。规范见 `插件开发规范.md`。

## 一、首期交付

1. **插件规范**：`plugin.json`（4 必填字段）+ 目录约定 + `window.itools` API 契约。文档 `插件开发规范.md`，类型 `plugins/itools.d.ts`。
2. **运行时**：后端扫描 `plugins/` → 解析清单 → 注入搜索索引；前端搜到关键字出插件磁贴 → 回车经自定义协议加载插件页 + 注入 `window.itools`。
3. **示例插件**（3 个，AI few-shot 参考）：`base64`（剪贴板编解码）、`json-format`（格式化/压缩）、`password`（`crypto` + `itools.db` 持久化设置）。
4. **热重载**：托盘「重新加载插件」即时重扫，改/生成插件无需重启（`rescan_plugins` / `PluginRegistry::reload`）。
5. **插件管理页**（管理中心导航「插件管理」）：列出已装插件、启用/禁用（是否参与主搜索）、删除、一键重载；禁用状态持久化在 `settings.disabled_plugins`。

## 二、架构与关键文件

**后端**（`src-tauri/src/plugin/`）
- `mod.rs`：清单解析/校验、`scan_plugins`/`expand_commands`、`PluginCommand`（关键字模糊 + regex 命中）、`serve`（`itplugin://` 协议，路径穿越校验 + CSP 头）、`PluginRegistry`、`plugins_root`（逐级上溯定位）。
- `commands.rs`：`open_plugin_window`（async 建窗规避死锁 + `on_navigation` 拦外链）+ 16 个 `plugin_*` 白名单命令。
- `bridge.js`：注入插件页的 `window.itools` 门面（`include_str!`）。
- 接入：`search/mod.rs`（plugins 池 + `set_plugin_commands` + query pass）、`lib.rs`（注册协议 + setup 扫描注入 + 命令注册）、`launch.rs`（`spawn_program`）。
- `build.rs`：`AppManifest::commands` 把 43 个命令纳入 ACL。
- `capabilities/default.json`（main/admin 授全部主程序命令）、`capabilities/plugin.json`（plugin 窗口只授 16 个 `plugin_*`）。

**前端**：`types.ts`（`plugin` 类型）、`main.ts`（`renderSearchGrid` 插件分区 + `execCell` 分流 `open_plugin_window`）。

## 三、安全模型（首期已加固）

> ⚠️ **Tauri v2 铁律**：`#[tauri::command]` 注册的 App 命令**默认对所有窗口全开**，capability 只约束 core/plugin 命令。故必须 `build.rs` 的 `AppManifest::commands` 把命令纳入 ACL 才能按窗口限制。（本项目一开始踩了这个坑，审查抓出、已修。）

- **命令 ACL**：全部 App 命令纳入 ACL；`plugin.json` **只**授 16 个 `plugin_*`，插件页**无法**调 `execute`/`save_settings`/`run_launch_item` 等（即无法经 `cmd:` 前缀执行任意命令）。
- **读写沙盒**：`readFile`/`writeFile` 均限 `<localAppData>/itools/plugin-data/<name>/files/`，组件白名单（Normal/CurDir）拒盘符/根/`..`，写前 canonicalize 复核。
- **执行链收敛**：`runCommand` 默认**关** + `program+args` 直 spawn（无 shell）；`openExternal` 仅 http/https/mailto；`openPath` 白名单（文档/图片/媒体/文件夹）+ 归一化尾点 + 拒 UNC。
- **外泄防护**：所有插件页**恒用严格 CSP**（`connect-src 'self'` 挡 fetch/xhr/img 外联 + `form-action 'self'` + `frame-ancestors 'none'` 禁被框）+ `on_navigation` 限 itplugin 源（挡顶层导航外泄）。
- **能力授权**：`runCommand`/`network` 需 `plugin.json.permissions` 声明 + 用户在「插件管理」按插件授权。**联网走原生代理 `itools.fetch`**（后端按【当前活动插件】授权实时判定）而非 CSP 放开——因所有插件同源，per-document CSP 会被同源 iframe 借道绕过（审查抓出的 HIGH，已根治；教训记 L2）。

## 五、首期范围与已知限制

- **触发**：关键字 + regex + `text`（任意输入命中）；`files/img` 仅解析不触发。关键字**只能裸字符串**（写成 `{type:keyword}` 对象会被静默忽略，日志有告警）。
- **加载**：启动时扫描一次；改/生成插件后点托盘「重新加载插件」即时生效（已支持热重载）。定位：dev 用项目 `plugins/`，打包用可写 `%LOCALAPPDATA%\itools\plugins`（首启从随包 `resource_dir/plugins` 播种内置示例）。
- **能力**：`readFile`/`writeFile` 仅沙盒；`openPath` 白名单外的文件类型拒开；`runCommand`(执行程序) 与 `network`(联网) 需 `plugin.json.permissions` 声明 + 用户在「插件管理」按插件授权后才可用；OS 原生通知未接（`notify` 落日志）。
- **打包分发**：插件目录为项目本地，未随安装包分发（`plugins_root` 已能上溯定位，但安装版需另配 `ITOOLS_PLUGINS_DIR` 或后续加 resource 打包）。
- **审查记录**：两轮多维对抗审查共确认并修复 14 处（3 critical：前端丢弃插件结果 / App 命令 ACL 全开 / runCommand 默认 RCE；2 high：读写沙盒逃逸 / 任意读外泄；及 openPath 绕过、CSP 导航、契约漂移等）。
