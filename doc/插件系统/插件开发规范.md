# iTools 插件开发规范 v1

> 面向 **AI 生成**：本规范就是给 Claude/MetaBot 照着一次生成可用插件的契约。
> 插件是 **给人用的页面插件**（关键词触发 → 打开 HTML 面板），但 **由 AI 在外部生成** 后放进项目 `plugins/` 目录，iTools 启动时自动扫描加载。

## 一、插件长什么样

一个插件 = `plugins/` 下的一个目录，**目录名必须等于 `plugin.json` 的 `name`**：

```
plugins/<name>/
├── plugin.json     # 唯一必需的清单
├── index.html      # UI 入口（约定文件名，勿改）
├── logo.png        # 图标（可选；无则用兜底字形）
└── assets/         # 其它静态资源（可选，index.html 里相对引用）
```

只要目录里同时有 `plugin.json` 和 `index.html` 就会被加载；坏插件（清单解析失败/缺文件）只告警跳过，不影响其它插件与主程序。

## 二、plugin.json

必填仅 4 项，其余可缺省：

```json
{
  "name": "base64",
  "version": "1.0.0",
  "description": "Base64 编码 / 解码",
  "features": [
    { "code": "main", "explain": "Base64 编码 / 解码", "cmds": ["base64", "b64", "编码"] }
  ]
}
```

| 字段 | 必填 | 说明 |
|---|---|---|
| `name` | ✅ | 唯一 id，小写字母数字连字符；**必须等于目录名**，同时是协议路径段 |
| `version` | ✅ | 语义化版本，如 `1.0.0` |
| `description` | ✅ | 一句话描述，进入搜索结果副标题 |
| `features` | ✅ | 功能命令数组，≥1 |
| `author` | — | 作者，缺省空 |
| `icon` | — | 图标文件名，缺省 `logo.png` |
| `permissions` | — | 声明所需的高危能力数组，如 `["runCommand","network"]`；**用户在「插件管理」按插件授权后才可用**（未声明/未授权则该能力调用被拒） |

**feature 对象**：

| 字段 | 必填 | 说明 |
|---|---|---|
| `code` | ✅ | 插件内唯一；进入插件时经 `onEnter(info)` 的 `info.code` 回传，用于区分是哪个功能 |
| `explain` | — | 功能说明，进入搜索结果标题（缺省用 `description`） |
| `cmds` | ✅ | 触发方式数组，≥1 |

**cmd 触发方式**（首期支持前两种）：

- **关键字**（字符串直配）：`"base64"` —— 搜索框输入该词（支持模糊）即命中。
- **regex**（正则）：`{ "type": "regex", "match": "^https?://" }` —— 输入匹配正则即命中（`match` 写正则源串，不要 `/.../` 包裹）。
- **text**（任意输入）：`{ "type": "text" }` —— 任意非空输入都命中该插件（排在关键字/应用之后，低优先级），进入时用户输入经 `onEnter` 的 `info.query` 传入（翻译 / 搜索 / 计算类插件用）。
- `{ "type": "files", "ext": ["png"] }` / `{ "type": "img" }` —— **首期仅解析、暂不触发**（向前兼容占位）。

> ⚠️ 关键字**只能写裸字符串**（如 `"base64"`）。写成 `{ "type": "keyword", "label": "base64" }` 这种对象形式**不受支持、会被静默忽略**（该 feature 将搜不到，仅日志有告警）。

## 三、index.html 与 window.itools

`index.html` 是一个普通网页（可内联 `<style>`/`<script>`，也可引用同目录 `assets/`）。iTools 在页面任何脚本前注入 `window.itools` 门面，**所有系统能力都经它调用**（拿不到 Node/完整 Tauri API）。类型定义见 `plugins/itools.d.ts`。

### 生命周期

```js
window.itools.onEnter((info) => {
  // info = { code, type, query }；进入插件即触发（读剪贴板、初始化都放这）
  // info.code 是命中的 feature.code
});
window.itools.onExit(() => { /* 页面卸载/隐藏 */ });
```

### API 清单

| 分类 | 方法 | 说明 |
|---|---|---|
| 窗口 | `hide()` / `exit()` / `setHeight(px)` | 隐藏 / 关闭 / 设高度（宽不变） |
| 剪贴板 | `copyText(s)` / `readText()` | 写 / 读剪贴板文本 |
| 文件 | `readFile(path)` / `writeFile(path, content)` | **均限插件沙盒**，path 为相对路径（禁绝对与 `..`）；读写对称，可持久化小文件 |
| 系统 | `openExternal(url)` / `openPath(path)` / `notify(body)` / `runCommand(program, args?)` | 开网址(仅 http/https/mailto) / 开路径(拒可执行文件) / 通知(**当前落日志，非系统通知**) / 执行程序(program+参数数组，不经 shell，**默认禁用**) |
| 存储 | `db.get(k)` / `db.set(k,v)` / `db.remove(k)` / `db.keys(prefix?)` | 按插件隔离的 KV，值自动 JSON 序列化，**纯本地**（**推荐用它做本地持久化**） |
| 账号态 | `account.state()` / `account.isLoggedIn()` | 只读：`{loggedIn,cloudConfigured,syncEnabled}`，判断是否登录云账号 |
| 同步数据 | `data.get/set/remove/keys` / `data.sync()` | 本地优先 + 可云同步；`sync()` 未登录/未接云端时诚实返回 `reason`，数据留本地 |
| UI | `showToast(msg)` | 轻量提示（同步，无需 await） |
| 平台 | `platform.{isWindows,isMacOS,isLinux,isDev}` | 平台标志 |

除 `showToast` 与 `platform` 外均返回 `Promise`。

### 插件设置（settings.json）

插件目录可放一个可选文件 `settings.json` 声明有哪些可配置项，iTools 就在「插件管理 → 点开插件 → 设置」tab **自动渲染设置界面并保存用户改动**（无 `settings.json` 则诚实显示「无可配置项」）。插件运行时用 `itools.settings`（**只读**）读取生效：`settings.get(k)` 读单项（不存在返回 `null`）、`settings.all()` 读全部对象、`settings.onChange(cb)` 在用户改设置时回调；读到的**值 = schema 默认 + 用户覆盖**。设置值只由用户在管理中心改，插件自己的业务状态仍用 `db`/`data`，不塞进 settings。控件类型与 `settings.json` 完整写法见 `skills/itools-plugin-dev/references/plugin-settings-spec.md`。

### 高危能力与授权

两类能力**默认禁用**，需在 `plugin.json.permissions` 声明 + 用户在「插件管理」页按插件授权后才可用：

- **`runCommand`**：`itools.runCommand(program, args?)` 执行程序（不经 shell）。未授权调用返回错误。
- **`network`**：插件页浏览器侧恒被严格 CSP 掐断外联（`connect-src 'self'`）——所有插件同源，靠 CSP 区分会被同源 iframe 借道绕过，故联网**不走浏览器 fetch**，而走原生代理 **`itools.fetch(url, init?)`**（http/https，返回 `{status, ok, body}` 文本）；门禁在原生层按【当前活动插件】的授权**实时**判定，撤销即时生效、无法被别的插件借道。

声明示例：`"permissions": ["network"]`。未声明/未授权的插件，`runCommand` 恒被拒、`itools.fetch` 恒被拒。

### 最小示例

```html
<!DOCTYPE html><html><body>
<textarea id="io"></textarea>
<button id="go">编码</button>
<script>
  const io = document.getElementById("io");
  window.itools.onEnter(async () => { io.value = await window.itools.readText(); io.focus(); });
  document.getElementById("go").onclick = () => {
    io.value = btoa(unescape(encodeURIComponent(io.value)));
    window.itools.copyText(io.value);
    window.itools.showToast("已编码并复制");
  };
</script>
</body></html>
```

完整参考实现见 `plugins/base64/`（UTF-8 安全编解码 + 剪贴板 + Toast + Esc 收起）。

## 四、加载与调试

- **放置**：把插件目录放进项目 `plugins/`（可用环境变量 `ITOOLS_PLUGINS_DIR` 覆盖根目录）。
- **加载/热重载**：iTools 启动时扫描一次；**新增/修改插件后，点托盘图标 →「重新加载插件」即时生效，无需重启**（AI 生成插件后立刻可测）。
- **触发**：主搜索栏输入 feature 的关键字 → 出现插件磁贴（图标=logo，副标题=描述）→ 回车打开插件页。
- **日志**：加载情况与 `notify` 都写 exe 同目录的 `itools.log`（搜「插件」）。
- **参考示例**：`plugins/` 下有 `base64`（剪贴板编解码）、`json-format`（格式化）、`password`（`db` 持久化设置）三个可直接照抄的样例。

## 五、给 AI 的生成清单（Checklist）

生成一个插件时逐条对齐：

1. 目录名 == `plugin.json.name`（小写连字符）。
2. `plugin.json` 含 name/version/description/features，且每个 feature 有唯一 `code` 与 ≥1 个 `cmds`。
3. `index.html` 存在，自包含（内联 CSS/JS 最省事），只用 `window.itools` 提供的能力。
4. 需要系统能力时对照上表选方法；`writeFile` 只写相对路径；别假设有 Node/`window.__TAURI__`。
5. 有 `logo.png` 更好（128×128 圆角方块风格），没有也能跑。
6. UI 适配深浅色（`prefers-color-scheme`），进入时 `onEnter` 里做初始化。

## 六、安全边界（实现者视角）

插件是不可信代码（自动加载、无签名审批），安全模型如下：

- **命令 ACL**：Tauri v2 默认允许任意窗口调用所有 `invoke_handler` 命令。故 `build.rs` 用 `AppManifest::commands` 把全部 App 命令纳入 ACL，再按窗口显式授权——`capabilities/plugin.json` **只**授予 16 个 `plugin_*` 白名单命令；插件页**无法**调用 `execute`/`save_settings` 等主程序命令（`capabilities/default.json` 才授予它们给 main/admin）。
- **读写沙盒**：`readFile`/`writeFile` 均限 `<localAppData>/itools/plugin-data/<name>/files/`，只收相对路径，按组件白名单（Normal/CurDir）拒绝盘符/根/`..`（修 Windows `is_absolute` 不认 `/foo`、`C:foo` 的绕过），写前再 canonicalize 复核父目录仍在沙盒内。
- **执行链收敛**：`runCommand` 默认关（`ALLOW_RUN_COMMAND=false`）且改走 `program+args` 直接 spawn（不经 `cmd.exe`，无 shell 注入）；`openExternal` 仅 http/https/mailto；`openPath` 拒 `cmd:` 前缀与可执行/脚本扩展名——堵住经 `open_detached` 的 `cmd:` 分支执行命令的旁路。
- **协议与外联**：自定义协议 `serve` 做路径穿越校验（canonicalize 后必须仍在插件目录内）；插件页响应带 CSP（允许内联脚本/样式，但 `connect-src 'self'` 掐断对外网络，防数据外泄）。
- 全局 `tauri.conf.json` 的 `csp` 仍为 null（不影响主窗口既有内联资源）；插件页的 CSP 由协议响应头单独下发，作用域仅插件页。
